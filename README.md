# Frank Andriulli’s Cybersecurity Journey

This repository documents my transition from enterprise IT support into cybersecurity.

The objective is to demonstrate continuous learning, hands-on technical development, security-focused problem solving, and the ability to document technical work clearly.

## Areas of Development

* Security and risk management
* Windows Server security
* Active Directory and identity security
* Security monitoring and event analysis
* Incident response
* Vulnerability management
* Security governance and controls
* CISSP preparation
* Professional cybersecurity development

## Repository Structure

### CISSP Study Notes

Summaries, diagrams, practice scenarios, and lessons covering the eight CISSP domains.

### Windows Security Labs

Hands-on exercises involving Windows Server, Active Directory, DNS, Group Policy, auditing, account security, and event logs.

### Incident Response Labs

Exercises involving alert investigation, evidence collection, containment decisions, root-cause analysis, and recovery recommendations.

### IAM Labs

Projects involving user provisioning, group membership, role-based access control, access reviews, privileged accounts, and account lifecycle management.

### Risk Assessments

Structured assessments documenting assets, threats, vulnerabilities, likelihood, business impact, existing controls, and recommended treatments.

### Career Development

Milestones, completed training, technical reflections, and lessons from my transition into cybersecurity.

## Documentation Standard

Each project will explain:

1. The business or security objective
2. The lab environment
3. The tools and technologies used
4. The procedure performed
5. The evidence collected
6. The findings
7. The security and business impact
8. The recommended remediation
9. What I learned
10. How the exercise relates to a cybersecurity analyst role

## Current Milestone

Building and securing an isolated Windows Server and Active Directory environment using Oracle VirtualBox.
